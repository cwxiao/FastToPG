# FastDBConvert 工具介绍

## 1. 工具概述

FastDBConvert 是一个用于 MySQL 迁移到 PostgreSQL 的离线迁移工具。它将“结构迁移”和“数据迁移”拆分执行，便于控制风险和定位问题。

工具提供两种使用方式：

- GUI：适合人工操作、实时查看日志与进度。
- CLI：适合批处理、脚本化和自动化执行。

当前版本支持：

- 整库迁移。
- 单表/多表迁移。
- 选表模式下的目标表定向清理。

## 2. 技术组成

整体采用“引擎 + 编排”的技术结构：

1. 结构迁移引擎：pgloader
   - 负责 MySQL 到 PostgreSQL 的表结构、索引、序列等对象同步。
   - 通过模板化 load 文件动态渲染迁移脚本。

2. 数据迁移引擎：DataX
   - 按表执行数据抽取与写入。
   - 支持并行迁移、批量写入和分片键策略。

3. 编排控制层：Python
   - 统一处理配置加载、任务调度、日志输出、失败中断和资源清理。
   - 对外提供：
     - `pgloader_gui.py`（图形界面）
     - `pgloader_tool.py`（命令行）

4. 配置中心：`pgloader_tool.json`
   - 管理源库/目标库连接、迁移参数、并发参数、日志策略等。

## 3. 大概使用流程

### 3.1 准备阶段

1. 准备运行环境（Python、Java、Docker）。
2. 确认 DataX 目录可用（`datax/datax/bin/datax.py`）。
3. 配置 `pgloader_tool.json` 中的源库、目标库和迁移参数。

### 3.2 执行阶段

1. 选择迁移库（可单库或多库）。
2. 如需单表/多表迁移，先选择数据库，再选择表。
3. 执行“结构同步”：
   - 未选表：清理目标库 `public` 全部表后同步结构。
   - 选表：仅清理目标库对应选中表后同步结构。
4. 执行“数据同步”：按配置并发迁移数据。

### 3.3 校验阶段

1. 查看日志确认无失败表。
2. 在目标库抽查表数量、关键表行数和核心业务数据。
3. 根据结果决定是否继续下一批次迁移。

## 4. 总结

FastDBConvert 的核心价值是：用 pgloader 和 DataX 组合实现“结构/数据分离”的可控迁移流程，并通过 GUI/CLI 提供统一、易用的迁移操作能力。
